
we can select the underlying database engine

we can choose the database settings template based on the type of system that we want i.e we want our db for production, dev or for free tier

we can choose availability and durability

we can also choose db instances classes which includes
1. standard classes (includes m class instances) : balanced between memory and cpu
2. memory optimized (includes r and x class instance) : more memory compared to cpu
3. burstable clases (includes t class instance) : has a low baseline performance but can burst when needed


we can also enable deletion protection of database so that we don't accidentally delete an entire database
