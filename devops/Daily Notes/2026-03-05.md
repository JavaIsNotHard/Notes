## What I Learned Today

### Tech -
- in-place pod resizing (vertical scaling) without requiring pod disruption and downtime. This feature was introduced to kubernetes v1.35 production
- connection pooling in microservice architecture, each service having its own connection pool setting set for an RDS instance
	- this configuration can cause max connections reached errors. Why??
	- Every service was managing its own pool. So during traffic spikes, autoscaling or deployments, connections would suddenly increase. Not because the DB was overloaded in compute, but because total open connections across services kept adding up.
	- The solution was to use RDS proxy which would facilitate the connection pooling to itself rather than directly to the RDS instance
